# 低值易耗品管理

## 1. 背景与目标

### 为什么需要这个功能
当前 AMS 资产管理系统只管理固定资产（如电脑、打印机、设备等），但中小企业日常运营中大量使用的低值易耗品（如打印纸、墨盒、鼠标、键盘、文件夹、笔等）没有纳入管理。这些物品采购频繁、使用分散、难以追踪，常出现"用完才知道要买"或"库存堆积"的问题。

### 解决什么问题
- 低值易耗品（如办公用品、生产耗材）无系统化管理
- 无alking库存视图，采购决策缺乏数据依据
- 无出入库记录，消耗情况不透明
- 无库存预警，导致紧急采购或影响工作

### 预期效果
- 员工可在线申请领用易耗品，系统记录出库
- 管理员可管理入库、查看实时库存
- 库存低于阈值时自动触发通知
- 消耗报表辅助采购预算决策

## 2. 功能范围

### 包含哪些功能点
1. **易耗品基础数据管理**：易耗品分类（如办公用品、电子配件、生产耗材）、基础信息维护（名称、规格、单位、阈值）
2. **易耗品入库**：记录每次采购入库，含数量、日期、供应商
3. **易耗品出库（领用）**：员工提交领用申请，审批后记录出库
4. **实时库存台账**：按分类/仓库展示当前库存余量
5. **库存预警**：库存低于最低阈值时，向管理员发送通知
6. **消耗报表**：按月/季度统计各类易耗品消耗数量和金额

### 不包含哪些（明确边界）
- 不管理固定资产（已有资产模块）
- 不含采购申请/审批流程（仅记录已完成的入库）
- 不含供应商管理（入库时可选择/新建供应商，但不独立维护供应商档案）
- 不含 RFID/条码打印（与固定资产标签打印区分，易耗品用简易编号）

## 3. 用户故事

### 角色：员工
- 场景：需要领用打印纸、鼠标等易耗品
- 期望：在线提交领用申请，查看历史领用记录

### 角色：管理员（ADMIN / MANAGER）
- 场景：收到一批新采购的打印纸，需要入库
- 期望：快速录入入库单，系统自动更新库存

### 角色：管理员（ADMIN）
- 场景：某类易耗品库存即将用完
- 期望：收到系统预警通知，及时安排采购

### 角色：财务/经理
- 场景：需要了解本季度易耗品消耗情况
- 期望：查看消耗报表，辅助下季度采购预算

## 4. 详细需求

### 4.1 页面/组件需求

#### 后台页面（Admin / Manager）

**易耗品管理页面** (`/#/consumables`)
- 左侧分类树（办公用品 / 电子配件 / 生产耗材 等）
- 右侧：易耗品列表（名称、规格、单位、当前库存、阈值、状态）
- 操作：新增 / 编辑 / 删除 易耗品
- 库存状态标签：正常（绿色）/ 预警（红色）

**入库管理页面** (`/#/consumables/stock-in`)
- 入库单列表：入库单号、供应商、易耗品、数量、日期、操作人
- 新增进库单：选择易耗品 → 输入数量 → 选择供应商（可选）→ 确认入库
- 入库后自动更新库存

**出库管理页面** (`/#/consumables/stock-out`)
- 出库单列表：单号、易耗品、领用人、数量、日期、审批状态
- 员工提交出库申请 → 管理员审批通过 → 记录出库
- 支持批量审批

**库存台账页面** (`/#/consumables/inventory`)
- 实时库存列表，按分类筛选
- 低于阈值项高亮显示

**消耗报表页面** (`/#/consumables/report`)
- 按月/季度查看消耗汇总（数量 + 预估金额）
- 柱状图 / 折线图展示趋势
- 支持按分类筛选

#### 前台页面（员工）

**易耗品领用页面** (`/#/consumables/apply`)
- 可申请易耗品种类列表
- 填写领用数量 → 提交申请
- 查看我的申请记录及审批状态

### 4.2 API/后端需求

#### 新建实体

**Consumable（易耗品基础表）**
```
id: Long (PK)
name: String (名称)
category: String (分类：OFFICE_SUPPLIES / ELECTRONIC_PARTS / PRODUCTION_CONSUMABLES)
spec: String (规格型号，可空)
unit: String (单位：如个、箱、卷)
threshold: Integer (最低库存预警阈值，默认 10)
createdAt: LocalDateTime
updatedAt: LocalDateTime
deleted: Boolean (软删除)
```

**ConsumableStock（库存台账表）**
```
id: Long (PK)
consumableId: Long (FK → Consumable)
currentStock: Integer (当前库存)
updatedAt: LocalDateTime
```

**ConsumableRecord（出入库记录表）**
```
id: Long (PK)
consumableId: Long (FK → Consumable)
type: ConsumableRecordType (IN / OUT)
quantity: Integer (数量)
relatedUserId: Long (操作人/领用人 FK → Employee)
supplier: String (供应商，入库时可选)
remark: String (备注)
createdAt: LocalDateTime
```

#### 新建 Enum
```java
public enum ConsumableCategory {
    OFFICE_SUPPLIES("办公用品"),
    ELECTRONIC_PARTS("电子配件"),
    PRODUCTION_CONSUMABLES("生产耗材");
}

public enum ConsumableRecordType {
    IN("入库"),
    OUT("出库");
}
```

#### 新建 Repository
- `ConsumableRepository` — 基础 CRUD + `findByDeletedFalse()` + 按分类查询
- `ConsumableStockRepository` — 按 consumableId 查询库存
- `ConsumableRecordRepository` — 按类型/时间范围查询记录

#### 新建 Service
- `ConsumableService` — 基础 CRUD
- `ConsumableStockService` — 入库（+库存）、出库（-库存，校验库存充足）、库存预警检查

#### 新建 Controller
- `ConsumableController` — CRUD API
- `ConsumableStockController` — 入库、出库 API（含审批流）
- `ConsumableReportController` — 消耗报表 API

#### 通知触发
- 库存低于阈值时，创建 NotificationType 为 `SYSTEM` 的通知，推送给 ADMIN

#### API 清单

| Method | Endpoint | 描述 |
|--------|----------|------|
| GET | /api/consumables | 列表（含分类筛选） |
| POST | /api/consumables | 新增易耗品 |
| PUT | /api/consumables/{id} | 编辑易耗品 |
| DELETE | /api/consumables/{id} | 删除易耗品 |
| GET | /api/consumables/{id}/stock | 查询库存 |
| POST | /api/consumables/stock-in | 入库 |
| POST | /api/consumables/stock-out | 出库（需审批） |
| GET | /api/consumables/records | 出入库记录列表 |
| GET | /api/consumables/report/consumption | 消耗报表 |

### 4.3 数据结构/字段变更

- 新建 3 张表：`consumable` / `consumable_stock` / `consumable_record`
- 资产表 `asset` 无变更（低值易耗品独立管理）
- 通知表可复用（NotificationType.SYSTEM）

## 5. 验收标准

### 功能上线判定条件
1. 管理员可在后台新增/编辑/删除易耗品基础数据
2. 管理员可录入入库单，库存自动增加
3. 员工可提交领用申请，管理员审批后库存自动扣减
4. 库存低于阈值时，NotificationBell 显示红色预警通知
5. 消耗报表页面可按月/季度查看消耗柱状图

### 验证方法
- 管理员新增易耗品"打印纸"，阈值为 5，入库 10 张 → 库存显示 10
- 员工提交领用 3 张打印纸 → 审批通过后 → 库存变为 7
- 再领用 3 张（剩余 4，低于阈值 5）→ 审批通过后 → 库存变为 4，触发系统通知
- 报表页面：查看本月消耗，显示打印纸消耗 6 张

## 6. 优先级

- **P1（重要）**
- 理由：中小企业日常运营中高频使用，是固定资产管理的必要补充；库存预警可以显著减少紧急采购，提升团队效率。
- 实施建议：可分期交付，第一期先实现基础 CRUD + 入库/出库手动记录，第二期再接入审批流 + 预警通知 + 报表。
