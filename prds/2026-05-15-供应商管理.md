# PRD: 供应商管理

## 1. 背景与目标

### 为什么需要这个功能
当前 AMS 系统通过 `asset.supplier` 字段记录供应商名称，但存在以下问题：
- 多个资产可能采购自同一供应商，但信息分散重复
- 无供应商联系方式（电话、邮箱、地址），需要时需手动查找
- 无供应商评分/历史记录，无法评估供应商质量
- 采购易耗品或维修服务时，也需要频繁选择/新建供应商

### 解决什么问题
- 供应商信息重复录入，维护成本高
- 供应商联系方式散落各处，查找困难
- 无法追溯与供应商的历史交易（采购、维修次数）
- 难以评估供应商服务质量

### 预期效果
- 管理员维护统一的供应商档案（名称、联系人、电话、邮箱、地址、类型）
- 资产和易耗品入库时可从供应商列表选择
- 维修记录中可关联供应商，并记录维修效果评分
- 供应商详情页展示与该供应商的所有交易历史

---

## 2. 功能范围

### 包含哪些功能点
1. **供应商档案管理**：增删改查供应商基本信息
2. **供应商类型**：设备供应商 / 易耗品供应商 / 维修服务商 / 多元化供应商
3. **供应商联系管理**：联系人、联系电话、邮箱、地址
4. **供应商资产关联查询**：查看某供应商提供的所有资产
5. **供应商维修历史**：查看某供应商提供的所有维修服务记录
6. **供应商评级**：好评/中评/差评，供采购参考

### 不包含哪些（明确边界）
- 不包含供应商招标/比价流程
- 不包含供应商合同管理（合同管理作为独立模块）
- 不包含供应商财务往来（付款/结算与财务系统集成）

## 3. 用户故事

### 角色：管理员（ADMIN / MANAGER）
- 场景：需要新增一个资产供应商（某电脑厂商）
- 期望：在供应商管理页面新增供应商档案，填写联系人、电话、公司地址

### 角色：资产管理员
- 场景：采购新电脑，入库时选择供应商
- 期望：从下拉列表选择已有供应商，而非手动输入名称

### 角色：维修管理员
- 场景：设备需要维修，联系第三方维修商
- 期望：在维修记录中选择维修商，维修完成后可对该维修商打分

### 角色：经理
- 场景：需要评估年度供应商表现
- 期望：查看供应商列表，了解每家供应商本年度提供的资产数量和维修次数

## 4. 详细需求

### 4.1 后端

#### Entity

**Supplier（供应商档案）**
```
id: Long (PK)
name: String (供应商名称，唯一)
type: SupplierType (EQUIPMENT / CONSUMABLE / MAINTENANCE / MULTI)
contactPerson: String (联系人，可选)
contactPhone: String (联系电话，可选)
contactEmail: String (邮箱，可选)
address: String (地址，可选)
rating: Integer (评分 1-5，可选)
remark: String (备注)
status: SupplierStatus (ACTIVE / INACTIVE)
createTime: LocalDateTime
updateTime: LocalDateTime
deleted: Boolean (软删除)
```

#### Enum

```java
public enum SupplierType {
    EQUIPMENT("设备供应商"),
    CONSUMABLE("易耗品供应商"),
    MAINTENANCE("维修服务商"),
    MULTI("多元化供应商");
}

public enum SupplierStatus {
    ACTIVE, INACTIVE
}
```

#### API

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | /api/suppliers | 分页列表（支持 name、type、status 筛选） |
| POST | /api/suppliers | 新增供应商 |
| PUT | /api/suppliers/{id} | 编辑供应商 |
| DELETE | /api/suppliers/{id} | 删除（软删除）供应商 |
| GET | /api/suppliers/{id} | 供应商详情 |
| GET | /api/suppliers/{id}/assets | 该供应商提供的所有资产 |
| GET | /api/suppliers/{id}/maintenance-records | 该供应商的维修记录 |
| GET | /api/suppliers/{id}/consumables | 该供应商供应的易耗品 |

#### 关联资产查询逻辑
- `asset.supplier` 字段存储的是供应商名称，与 `supplier.name` 匹配
- 查询时模糊匹配或精确匹配

### 4.2 前端

| 页面 | 路由 | 说明 |
|------|------|------|
| SupplierList | /# /suppliers | 供应商列表（类型Tab + 筛选） |
| SupplierForm | /# /suppliers/form | 新增/编辑供应商表单 |
| SupplierDetail | /# /suppliers/:id | 供应商详情 + 关联资产/维修记录 |
| SupplierSelect | 通用选择器 | 资产入库/易耗品入库/维修记录中的供应商选择组件 |

### 4.3 菜单
- 「供应商管理」菜单项（Manager/Admin 可见）

---

## 5. 验收标准

1. 管理员可新增/编辑/删除供应商档案
2. 供应商列表页支持按类型/状态筛选
3. 资产入库表单中，供应商字段改为下拉选择（可搜索）
4. 维修记录中，维修商字段改为下拉选择
5. 供应商详情页显示该供应商提供的资产数量和维修次数
6. `mvn compile` 成功，`npm run build` 成功

---

## 6. 优先级

**P2（有益）** — 供应商管理可提升采购和维修业务效率，是资产管理系统的重要补充
