# PRD: 资产保险管理

## 1. 背景与目标

### 为什么需要这个功能
当前 AMS 系统已实现维保到期提醒（warrantyEnd），但保险与维保是两个独立概念：
- **维保**：设备原厂或第三方提供故障维修服务
- **保险**：资产因意外（火灾/盗窃/意外损坏）造成损失的赔付保障

 enterprises 需要对高价值资产（如服务器、精密仪器、车辆等）单独购买商业保险。保险信息若仅记录在 Excel 或纸质文档中，容易遗漏续保，导致资产出事时无法索赔。

### 解决什么问题
- 高价值资产保险信息无系统化管理
- 保险到期无提醒，错失续保窗口
- 保险索赔记录无存档
- 无法统计企业保险覆盖率

### 预期效果
- 管理员可为指定资产关联保险单（保单号、保险公司、保险类型、保险金额、起止日期）
- 保险到期前 N 天自动推送续保提醒
- 保险索赔记录可追溯
- 仪表盘展示保险即将到期的资产

---

## 2. 功能范围

### 包含哪些功能点
1. **保险单管理**：增删改查保险单（保单号、保险公司、保险类型：财产险/综合险/盗抢险、投保金额、保险期间）
2. **资产保险关联**：一个资产可关联一个或多个保险单（主要保险 + 附加保险）
3. **保险到期提醒**：与维保到期提醒机制一致，提前 N 天推送系统通知
4. **保险索赔记录**：记录保险报案、索赔金额、赔付状态、赔付日期
5. **仪表盘视图**：展示保险即将到期的资产数量和列表

### 不包含哪些（明确边界）
- 不包含保险费用报销流程（与财务系统集成）
- 不包含保险公司主数据管理（仅记录本次投保的保险公司名称）
- 不包含自动计算保险费率

---

## 3. 用户故事

### 角色：管理员（ADMIN / MANAGER）
- 场景：采购了一批价值 50 万的服务器，需要购买财产保险
- 期望：在资产详情页关联保险单，填写保单号、保险公司、保障期限
- 后续：保险快到期时收到系统提醒，及时续保

### 角色：管理员
- 场景：某服务器因雷击损坏，需要走保险索赔
- 期望：在该资产的保险记录中新增一条索赔记录，登记索赔金额和赔付状态

### 角色：财务/经理
- 场景：季度审查资产保险覆盖率
- 期望：查看报表，了解有多少资产有保险、保额分布、保单到期情况

## 4. 详细需求

### 4.1 后端

#### Entity

**AssetInsurance（资产保险单）**
```
id: Long (PK)
assetId: Long (FK → asset)
policyNo: String (保单号，唯一)
insuranceCompany: String (保险公司名称)
insuranceType: InsuranceType (PROPERTY / COMPREHENSIVE / THEFT)
insuredAmount: BigDecimal (投保金额)
premium: BigDecimal (保费，可选)
startDate: LocalDate (保险开始日期)
endDate: LocalDate (保险结束日期)
status: InsuranceStatus (ACTIVE / EXPIRED / CANCELLED)
remark: String (备注)
createTime: LocalDateTime
updateTime: LocalDateTime
```

**InsuranceClaim（保险索赔记录）**
```
id: Long (PK)
insuranceId: Long (FK → asset_insurance)
claimNo: String (索赔单号)
claimAmount: BigDecimal (索赔金额)
compensatedAmount: BigDecimal (赔付金额，可选)
claimDate: LocalDate (报案日期)
settledDate: LocalDate (结案日期，可选)
status: ClaimStatus (PENDING / SETTLED / REJECTED)
description: String (出险经过)
createTime: LocalDateTime
updateTime: LocalDateTime
```

#### Enum

```java
public enum InsuranceType {
    PROPERTY("财产险"),
    COMPREHENSIVE("综合险"),
    THEFT("盗抢险");
}

public enum InsuranceStatus {
    ACTIVE, EXPIRED, CANCELLED
}

public enum ClaimStatus {
    PENDING, SETTLED, REJECTED
}
```

#### API

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | /api/insurances | 分页列表（支持 assetId、status 筛选） |
| POST | /api/insurances | 新增保险单 |
| PUT | /api/insurances/{id} | 编辑保险单 |
| DELETE | /api/insurances/{id} | 删除保险单 |
| GET | /api/insurances/{id} | 详情 |
| GET | /api/insurances/asset/{assetId} | 获取某资产的所有保险单 |
| POST | /api/insurances/{id}/claims | 新增索赔记录 |
| GET | /api/insurances/claims | 索赔记录列表（支持分页、状态筛选） |
| PUT | /api/insurances/claims/{id} | 编辑索赔记录 |
| GET | /api/dashboard/expiring-insurance?days=30 | 仪表盘用：即将到期的保险 |

#### Service 层

**InsuranceNotificationService**
- `@Scheduled` 每日扫描保险到期日（与 WarrantyNotificationService 类似）
- 发送 ASSET_EXPIRING_INSURANCE 类型通知
- 去重机制（同一保险单同一批次只通知一次）

### 4.2 前端

| 页面 | 路由 | 说明 |
|------|------|------|
| InsuranceList | /# /insurance | 保险单列表（可按资产/状态筛选） |
| InsuranceForm | /# /insurance/form | 新增/编辑保险单表单 |
| InsuranceDetail | /# /insurance/:id | 保险单详情 + 索赔记录 |
| ClaimForm | /# /insurance/:id/claim | 新增索赔记录表单 |
| Dashboard 保险卡片 | 仪表盘 | 显示即将到期保险数量，点击展开列表 |

### 4.3 菜单
- 「保险管理」菜单项（Manager/Admin 可见）

## 5. 验收标准

1. 管理员可为资产新增保险单，保存后保险单列表显示
2. 保险到期前 30 天，NotificationBell 显示提醒，数量正确
3. 管理员可在保险单详情页新增索赔记录
4. 仪表盘「保险即将到期」卡片正确显示数量和列表
5. `mvn compile` 成功，`npm run build` 成功
