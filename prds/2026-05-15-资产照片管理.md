# PRD: 资产照片管理 (Asset Photo Management)

## 1. 背景与目标

**问题**: 当前 AMS 的资产记录仅包含文字信息（名称、编码、型号、购置日期等），无法通过照片进行视觉识别。在资产盘点、借还、巡检场景中，仅靠文字描述难以快速确认资产身份，导致盘点效率低、易混淆相近设备。

**目标**: 在现有资产管理体系上，增加资产照片的上传、存储、查看能力，实现"一物一图"的视觉化管理，提升盘点效率和资产识别准确度。

---

## 2. 功能范围

### In Scope
- 资产详情页显示主照片（一张主图）
- 新增/编辑资产时上传照片（支持 JPEG/PNG，单文件 ≤ 5MB）
- 资产列表页显示缩略图
- 照片单独查看（大图预览）
- 删除/替换已有照片
- 后端 Base64 存储（或 nginx 静态文件路径，二选一落地）

### Out of Scope
- 多图相册（仅一张主图）
- 拍照上传（仅文件选择器）
- 照片 EXIF 解析
- 照片水印/压缩

---

## 3. 用户故事

| Role | Scenario | Expected Outcome |
|------|----------|-----------------|
| ADMIN/MANAGER | 新增资产，添加照片 | 照片保存成功，详情页可见 |
| ADMIN/MANAGER | 盘点时查看资产照片 | 一眼确认资产身份，效率提升 |
| ADMIN/MANAGER | 编辑资产，替换照片 | 旧照片被覆盖，新照片显示 |
| ADMIN/MANAGER | 删除资产时 | 照片一并清理（或软删除资产时保留照片） |
| EMPLOYEE | 查看自己领用的资产 | 看到资产实拍图，更有归属感 |

## 4. 详细需求

### 4.1 后端

#### Entity
```java
// Asset 实体新增字段
@Column(name = "photo_url", length = 500)
private String photoUrl;  // Base64 或相对路径
```

#### API

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/assets/{id}/photo` | 上传/更新资产照片（multipart form） |
| GET | `/api/assets/{id}/photo` | 获取照片（返回图片或 Base64） |
| DELETE | `/api/assets/{id}/photo` | 删除照片 |
| GET | `/api/assets/{id}` | AssetDetailResponse 新增 photoUrl 字段 |

#### Storage Strategy
- **Phase 1 (simple)**: Base64 存储在 `photo_url` 列（无额外存储依赖）
- **Phase 2 (optional)**: nginx 静态文件目录，photo_url 存相对路径

### 4.2 前端

#### AssetForm（新增/编辑）
- 照片上传区域：拖拽或点击选择文件
- 预览已上传照片
- 支持 JPEG/PNG，5MB 限制，前端校验
- 上传成功后显示缩略图

#### AssetDetail（详情页）
- 照片展示区（大图，点击放大）
- 无照片时显示占位图（SVG 占位符）

#### AssetList（列表页）
- 列表新增一列"照片"：小缩略图或占位图标
